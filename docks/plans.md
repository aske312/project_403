# 🧩 Breakdown задач для технической команды (MVP мессенджера)

---
## 📌 1. Общий подход
Проект разбивается на независимые блоки:
* Backend
* Frontend
* DevOps (минимальный)
* Тестирование

Работа ведётся итерационно (по этапам MVP)

---
## 🧱 2. EPIC 1 — Подготовка проекта
### 🎯 Цель:
Развернуть базовую инфраструктуру разработки

### Backend:
* [x] Создать репозиторий
* [x] Инициализировать проект (FastAPI)
* [x] Настроить структуру проекта
* [x] Настроить конфигурацию (env)

### Frontend:
* [x] Создать проект (React)
* [x] Настроить базовую структуру

### DevOps:
* [x] Поднять PostgreSQL (локально / Docker)
* [x] Настроить переменные окружения
* [x] Настроить запуск проекта

### 🎯 Цель:

* [ ] Генерация JWT

* [x] Форма регистрации
* [x] Форма логина

* [ ] Connection Manager:

* [ ] Подключение к WebSocket
* [ ] Отправка сообщений

## 12. 📌 Definition of Ready (DoR)
Задача готова к работе, если:
* есть описание
* понятен результат
* нет блокеров

---
## 13. ✅ Definition of Done (DoD)

Задача завершена, если:
* работает функционально
* протестирована вручную
* интегрирована с системой
