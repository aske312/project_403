# Project_403 Roadmap

Актуализировано: 2026-05-04

## 1. Концепция продукта

Project_403 - это коммуникационная платформа для личного и командного общения. Мы строим продукт на пересечении трех привычных сценариев:

- Telegram: быстрые личные переписки, контакты, группы, каналы, мобильная логика общения.
- Slack: рабочие пространства, командные каналы, роли, администрирование, поиск по рабочему контексту.
- Discord: сообщества, голосовые комнаты, групповые звонки, живое присутствие пользователей.

Целевой результат: единое приложение, где пользователь может общаться лично, работать внутри команды, вести каналы, проводить звонки и управлять пространствами без перехода между разными сервисами.

## 2. Продуктовые принципы

- Сначала надежное ядро общения: аккаунт, пользователи, чаты, сообщения, realtime.
- Личная коммуникация и рабочие пространства должны развиваться как части одной модели, а не как два разных продукта.
- Админка и dev-инструменты остаются частью платформы, потому что проект быстро растет и нуждается в наблюдаемости.
- Каждая крупная возможность должна иметь backend API, frontend flow, базовую безопасность и проверяемый сценарий.
- Звонки, конференции и корпоративные возможности идут после стабильного текстового общения.

## 19. Управленческие риски

- Realtime и звонки нельзя начинать до стабильной модели сообщений, иначе придется переписывать события.
- Workspace лучше проектировать после личных и групповых чатов, чтобы не усложнить MVP раньше времени.
- Email confirmation, refresh token и session management нужно закрыть до публичного тестирования.
- Production deploy без миграций БД создаст риск потери или несовместимости данных.
- UI мессенджера должен стать основным экраном после входа, иначе продукт будет выглядеть как auth/admin demo, а не коммуникационная платформа.
